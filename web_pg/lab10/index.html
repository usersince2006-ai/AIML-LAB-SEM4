Experiment 10: Express.js Backend

Hello World API · Routing · Middleware · nodemon workflow

⚙️ Express.js Setup (server.js)

// server.js — Basic Express server const express = require('express'); const app = express(); const PORT = 3000; // Middleware: parse JSON request bodies app.use(express.json()); // Logging middleware (custom) app.use((req, res, next) => { console.log(`${req.method} ${req.url}`); next(); }); // Route: GET / → Hello World app.get('/', (req, res) => { res.json({ message: 'Hello, World!' }); }); // Route: GET /api → API info app.get('/api', (req, res) => { res.json({ status: 'running', version: '1.0.0', endpoints: ['/', '/api', '/api/greet'] }); }); // Route: GET /api/greet?name=Tony app.get('/api/greet', (req, res) => { const name = req.query.name || 'World'; res.json({ message: `Hello, ${name}!` }); }); // Start server app.listen(PORT, () => { console.log(`Server running on port ${PORT}`); });

📚 Key Concepts

  • Node.js: JavaScript runtime on server-side (V8 engine)
  • Express.js: Minimal framework for routing & middleware
  • Route: Maps URL + HTTP method → handler function
  • Middleware: Functions that run between request & response
  • nodemon: Auto-restarts server on file change during dev
  • res.json(): Sends JSON response with correct headers
  • req.query: URL query parameters (?key=value)
  • req.body: POST request body (needs express.json())
